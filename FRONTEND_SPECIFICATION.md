# UTM Shortener Pro - Frontend Specification

## Завдання для розробника фронтенду

Цей документ містить детальну специфікацію для розробки фронтенду UTM Shortener Pro. Всі вимоги, дизайн-системи та технічні деталі описані нижче.

---

## 1. Огляд проекту

**UTM Shortener Pro** — це веб-утиліта для маркетологів та SMM-спеціалістів, яка дозволяє швидко генерувати та скорочувати множину UTM-посилань для різних соціальних мереж та типів контенту в одному пакеті.

### Основні характеристики:
- Багатомовність: українська та англійська мови
- Дві теми оформлення: світла (з синіми відтінками) та темна (золото на синьому)
- Інтерактивна матриця вибору платформ та плейсментів
- Історія генерованих посилань з автоматичним видаленням через тиждень
- Управління API-ключами користувача
- Копіювання посилань в буфер обміну

### 4.2 Main Form Component

**Розташування**: Верхня частина основного контейнера

**Елементи**:

- **Мітка**: "Базова URL" (UK) / "Base URL" (EN)
- **Плейсхолдер**: "https://example.com/page"
- **Тип**: text input
- **Валідація**:
  - Обов'язкове поле
  - Має бути валідна URL (перевірка через `new URL()`)
  - Помилка: "Невалідна URL" (UK) / "Invalid URL" (EN)
  - Помилка відображається під полем червоним кольором
- **Ширина**: 100%
- **Padding**: 12px 16px
- **Бордер**: 2px solid, колір акценту при фокусі
- **Фон**: темніший за основний на 5%

#### 4.2.2 Campaign Name Input

- **Мітка**: "Назва кампанії" (UK) / "Campaign Name" (EN)
- **Плейсхолдер**: "Введіть назву кампанії" (UK) / "Enter campaign name" (EN)
- **Тип**: text input
- **Валідація**:
  - Обов'язкове поле
  - Максимум 255 символів
  - Помилка: "Невалідна назва кампанії" (UK) / "Invalid campaign name" (EN)
  - Помилка відображається під полем червоним кольором
- **Ширина**: 100%
- **Padding**: 12px 16px
- **Бордер**: 2px solid, колір акценту при фокусі
- **Фон**: темніший за основний на 5%

#### 4.2.3 Form Layout

- **Контейнер**: card (див. дизайн-систему)
- **Padding**: 30px
- **Margin bottom**: 30px
- **Gap між полями**: 20px
- **Flex layout**: column

**Код структури**:
```tsx
<div className="form-card">
  <div className="form-group">
    <label htmlFor="baseUrl">Базова URL</label>
    <input
      id="baseUrl"
      type="text"
      placeholder="https://example.com/page"
      value={baseUrl}
      onChange={(e) => setBaseUrl(e.target.value)}
      className="input"
    />
    {errors.baseUrl && <span className="error">{errors.baseUrl}</span>}
  </div>

  <div className="form-group">
    <label htmlFor="campaign">Назва кампанії</label>
    <input
      id="campaign"
      type="text"
      placeholder="Введіть назву кампанії"
      value={campaign}
      onChange={(e) => setCampaign(e.target.value)}
      className="input"
    />
    {errors.campaign && <span className="error">{errors.campaign}</span>}
  </div>
</div>
```

```
│  Instagram     [ ]     [ ]      [ ]                │
│  Threads       [ ]     [ ]      [ ]                │
├─────────────────────────────────────────────────────┤
│  [Select All]  [Clear All]                         │

```

#### 4.3.2 Платформи

| Код | Назва (UK) | Назва (EN) | Іконка |
|-----|-----------|-----------|--------|
| tg | Telegram | Telegram | 📱 |

| Код | Назва (UK) | Назва (EN) |
|-----|-----------|-----------|

| reels | Рилс | Reels |

#### 4.3.4 Функціональність

**Вибір комірки**:
- Клік на комірку: вибір/відміна вибору
- Активна комірка: фон кольору акценту, текст білий
- Неактивна комірка: прозорий фон, текст звичайний
- Плавний перехід при клікі (transition 0.2s)

**Select All**:
- Вибирає всі 15 комбінацій (5 платформ × 3 плейсменти)
- Текст: "Вибрати все" (UK) / "Select All" (EN)
- Кнопка: secondary style

**Clear All**:
- Відміняє вибір всіх комбінацій
- Текст: "Очистити все" (UK) / "Clear All" (EN)
- Кнопка: secondary style

**Валідація**:
- Кнопка "Generate" активна тільки якщо:
  - Base URL заповнена та валідна
  - Campaign Name заповнена та валідна
  - Вибрана принаймні одна комбінація
- Якщо умови не виконані: кнопка неактивна (opacity 0.5, cursor not-allowed)

#### 4.3.5 Стилізація матриці

- **Контейнер**: card
- **Padding**: 30px
- **Margin bottom**: 30px
- **Таблиця**: border-collapse, width 100%
- **Заголовок**: bold, колір акценту
- **Комірки**: 
  - Padding: 15px
  - Border: 1px solid, колір акценту з прозорістю 0.2
  - Cursor: pointer
  - User-select: none
- **Кнопки**: 
  - Gap: 10px
  - Margin top: 20px
  - Display: flex

**Код структури**:
```tsx
<div className="matrix-card">
  <h3>Платформи / Плейсменти</h3>
  <table className="matrix-table">
    <thead>
      <tr>
        <th></th>
        <th>Пост</th>
        <th>Історія</th>
        <th>Рилс</th>
      </tr>
    </thead>
    <tbody>
      {platforms.map((platform) => (
        <tr key={platform.code}>
          <td>{platform.name}</td>
          {mediums.map((medium) => (
            <td
              key={`${platform.code}-${medium.code}`}
              className={`matrix-cell ${isSelected(platform.code, medium.code) ? 'active' : ''}`}
              onClick={() => toggleSelection(platform.code, medium.code)}
            >
              ✓
            </td>
          ))}
        </tr>
      ))}
    </tbody>
  </table>
  <div className="matrix-actions">
    <button onClick={selectAll} className="btn btn-secondary">Вибрати все</button>
    <button onClick={clearAll} className="btn btn-secondary">Очистити все</button>
  </div>
</div>
```

### 4.4 Generate Button

**Розташування**: Під матрицею

**Характеристики**:
- **Текст**: "Генерувати посилання" (UK) / "Generate Links" (EN)
- **Тип**: primary button
- **Ширина**: 100%
- **Висота**: 50px
- **Шрифт**: 16px, bold
- **Стан**:
  - Активна: колір акценту, cursor pointer
  - Неактивна: сірий, opacity 0.5, cursor not-allowed
  - Hover: темніший колір акценту
  - Loading: показує спінер, текст "Генерування..."
- **Margin bottom**: 30px

**Функціональність**:
- Клік: відправляє запит на `/api/links/generate`
- Під час обробки: показує спінер
- При помилці: показує повідомлення про помилку
- При успіху: показує результати

**Код структури**:
```tsx
<button
  onClick={generateLinks}
  disabled={!isFormValid() || isLoading}
  className="btn btn-primary w-full h-12 text-lg font-bold"
>
  {isLoading ? (
    <>
      <span className="spinner spinner-md mr-2"></span>
      Генерування...
    </>
  ) : (
    'Генерувати посилання'
  )}
</button>
```

### 4.5 Results Section Component

**Розташування**: Під кнопкою Generate (з'являється після успішної генерації)

**Умови відображення**:
- Показується тільки якщо є результати
- Плавна поява (fade-in анімація)

```
┌────────────────────────────────────────────────────────────┐
│ РЕЗУЛЬТАТИ                                                 │
├────────────────────────────────────────────────────────────┤
│ Платформа | Плейсмент | Повне посилання | Скорочене       │
├────────────────────────────────────────────────────────────┤
│ Telegram  | Пост      | [link] [Copy]   | [Copy]          │
│ Facebook  | Історія   | [link] [Copy]   | [Copy]          │
```

#### 4.5.2 Колони таблиці

| Колона | Опис | Ширина |
|--------|------|--------|

| Повне посилання | Повна URL з UTM параметрами | 35% |
| Скорочене | Скорочене посилання від TinyURL | 35% |

#### 4.5.3 Функціональність кожного рядка

**Copy кнопка для повного посилання**:
- Іконка: 📋
- Клік: копіює повне посилання в буфер обміну
- Feedback: кнопка змінює текст на "Скопійовано!" на 2 секунди
- Колір: синій (темна тема) / синій (світла тема)

**Copy кнопка для скороченого посилання**:
- Іконка: 📋
- Клік: копіює скорочене посилання в буфер обміну
- Feedback: кнопка змінює текст на "Скопійовано!" на 2 секунди
- Колір: золотий (темна тема) / синій (світла тема)

**Статус посилання**:
- Success (зелений): посилання успішно згенеровано та скорочено
- Failed (червоний): помилка при скороченні
  - Показує повідомлення про помилку
  - Кнопка Retry для повторної спроби
  - Приклад помилки: "API ключ вичерпаний" або "Невалідна URL"

#### 4.5.4 Copy All кнопка

- **Розташування**: під таблицею
- **Текст**: "Копіювати все" (UK) / "Copy All" (EN)
- **Функціональність**: копіює всі скорочені посилання в буфер обміну в форматі:
  ```
  Telegram Post: https://tinyurl.com/abc123
  Facebook Story: https://tinyurl.com/def456
  ...
  ```
- **Feedback**: кнопка змінює текст на "Скопійовано!" на 2 секунди
- **Стиль**: primary button

#### 4.5.5 Стилізація

- **Контейнер**: card
- **Padding**: 30px
- **Margin bottom**: 30px
- **Таблиця**: 
  - Width: 100%
  - Border-collapse: collapse
  - Font-size: 14px
- **Заголовок таблиці**:
  - Background: колір акценту з прозорістю 0.1
  - Font-weight: bold
  - Padding: 15px
  - Border-bottom: 2px solid, колір акценту
- **Рядки таблиці**:
  - Padding: 15px
  - Border-bottom: 1px solid, колір акценту з прозорістю 0.2
  - Hover: background колір акценту з прозорістю 0.05
- **Посилання в таблиці**:
  - Truncate текст (ellipsis)
  - Cursor: pointer
  - Hover: підкреслення
  - Можна клікнути для копіювання

**Код структури**:
```tsx
<div className="results-card">
  <h3>Результати</h3>
  <div className="results-generated-at">
    Згенеровано: {new Date().toLocaleString(language === 'uk' ? 'uk-UA' : 'en-US')}
  </div>
  <table className="results-table">
    <thead>
      <tr>
        <th>Платформа</th>
        <th>Плейсмент</th>
        <th>Повне посилання</th>
        <th>Скорочене посилання</th>
      </tr>
    </thead>
    <tbody>
      {results.map((result, index) => (
        <tr key={index} className={`status-${result.status}`}>
          <td>{result.platform}</td>
          <td>{result.medium}</td>
          <td>
            <span className="link-text">{result.fullUrl}</span>
            <button onClick={() => copyToClipboard(result.fullUrl)} className="copy-btn">
              📋
            </button>
          </td>
          <td>
            {result.shortUrl ? (
              <>
                <span className="link-text">{result.shortUrl}</span>
                <button onClick={() => copyToClipboard(result.shortUrl)} className="copy-btn">
                  📋
                </button>
              </>
            ) : (
              <span className="error-text">{result.error}</span>
            )}
          </td>
        </tr>
      ))}
    </tbody>
  </table>
  <button onClick={copyAll} className="btn btn-primary mt-4">
    Копіювати все
  </button>
</div>
```

## 7. Обробка помилок та валідація

### 7.1 Валідація форми

| Поле | Правило | Повідомлення (UK) | Повідомлення (EN) |
|------|---------|------------------|------------------|
| Base URL | Обов'язкове, валідна URL | Невалідна URL | Invalid URL |
| Campaign | Обов'язкове, 1-255 символів | Невалідна назва кампанії | Invalid campaign name |
| Matrix | Мінімум 1 комбінація | Виберіть принаймні одну комбінацію | Select at least one combination |
| API Key | Мінімум 20 символів | Невалідний API ключ | Invalid API key |

### 7.2 Обробка помилок від API

| Код | Причина | Повідомлення (UK) | Дія |
|-----|---------|------------------|-----|
| 400 | Невалідні дані | Невалідні дані форми | Показати помилку під полем |
| 401 | Невалідний API ключ | API ключ невалідний | Показати помилку в модалі налаштувань |
| 422 | Невалідна URL | Невалідна URL для скорочення | Показати помилку в рядку результату + Retry |
| 429 | Rate limit | API ключ вичерпаний | Показати помилку + пропозиція замінити ключ |
| 500 | Серверна помилка | Помилка сервера | Показати помилку + Retry |

### 7.3 Retry механізм

- **Для посилань**: кнопка "Повторити" (UK) / "Retry" (EN) для кожного невдалого посилання
- **Для всіх**: кнопка "Повторити все" (UK) / "Retry All" (EN) для повторної спроби всіх невдалих посилань
- **Максимум спроб**: 3 спроби перед остаточною помилкою

## 8. Функціональні вимоги

### 8.1 Генерування посилань

1. Користувач заповнює форму (Base URL, Campaign Name)
2. Користувач вибирає комбінації платформ та плейсментів
3. Користувач клікає "Generate Links"
4. Система валідує дані
5. Система генерує UTM посилання для кожної комбінації
6. Система відправляє кожне посилання на TinyURL API
7. Система отримує скорочені посилання
8. Система відображає результати в таблиці
9. Система зберігає результати в історії

### 8.2 Копіювання посилань

1. Користувач клікає на кнопку "Copy" для окремого посилання
2. Система копіює посилання в буфер обміну
3. Система показує "Скопійовано!" на 2 секунди
4. Користувач може клікнути "Copy All" для копіювання всіх посилань
5. Система копіює всі посилання в форматі "Platform Medium: URL"

### 8.3 Управління налаштуваннями

1. Користувач клікає на ⚙️ в Header
2. Система відкриває модаль налаштувань
3. Користувач вводить або змінює API ключ
4. Система валідує ключ
5. Користувач клікає "Замінити ключ"
6. Система зберігає ключ в localStorage
7. Система показує "Налаштування збережені"

### 8.4 Управління мовою та темою

1. Користувач клікає на "UK" або "EN" в Header
2. Система змінює мову всього інтерфейсу
3. Система зберігає вибір в localStorage
4. Користувач клікає на 🌙 або ☀️ в Header
5. Система змінює тему всього інтерфейсу
6. Система зберігає вибір в localStorage

### 8.5 Управління історією

## 12. Примітки для розробника

1. **Персистентність**: Всі налаштування та історія зберігаються в localStorage. Це означає, що користувач не потребує реєстрації для базового використання.

2. **API Key**: Користувач вводить свій власний TinyURL API ключ. Ключ зберігається в localStorage браузера користувача, не на сервері.

3. **Розширюваність**: Архітектура дозволяє легко додавати нові платформи та плейсменти без змін у коді компонентів. Просто додайте нові записи в базу даних.

4. **Мультимова**: Всі рядки повинні бути в `src/i18n/translations.ts`. Не жорстко кодуйте рядки в компонентах.

5. **Теми**: Використовуйте класи Tailwind для теми. Темні кольори автоматично застосовуються при класі `dark` на HTML елементі.

6. **Responsive**: Тестуйте на мобільних пристроях. Матриця та таблиці повинні мати горизонтальний скрол на мобілі.

7. **Performance**: Уникайте непотрібних re-renders. Використовуйте `React.memo` для компонентів, які часто перемальовуються.

8. **Error Handling**: Завжди показуйте користувачу зрозумілі повідомлення про помилки. Не показуйте технічні деталі.

9. **Loading States**: Показуйте спінер під час обробки. Деактивуйте кнопки під час завантаження.

10. **Copy Feedback**: Дайте користувачу зворотний зв'язок при копіюванні. Змініть текст кнопки на "Скопійовано!" на 2 секунди.
